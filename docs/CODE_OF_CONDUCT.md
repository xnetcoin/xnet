# Code of Conduct

## Our Commitment

We are committed to maintaining a professional, open, and harassment-free environment for everyone who participates in the XNET project — contributors, validators, developers, and community members alike, regardless of background, experience level, nationality, or identity.

## Expected Behavior

All participants are expected to:

- Use clear, professional, and respectful language
- Give and accept constructive criticism in good faith
- Focus on what is technically correct and best for the protocol
- Respect differing engineering judgments and viewpoints
- Report genuine issues through appropriate channels

## Unacceptable Behavior

The following will not be tolerated in any project space — GitHub issues, pull requests, community channels, or events:

- Harassment, threats, or personal attacks of any kind
- Discriminatory language or behavior
- Publishing private information of others without consent
- Deliberate disruption of technical discussions
- Using public forums to lobby for specific token-related outcomes or governance positions

## Protocol Neutrality

XNET software is a tool. The project does not take positions on how individual networks built with it should govern themselves. Technical forums and issue trackers are for engineering discussion only — not for expressing preferences about on-chain governance decisions, tokenomics changes, or community politics.

## Enforcement

Violations may be reported to the maintainers via the contact listed in [SECURITY.md](SECURITY.md) or by opening a private GitHub issue. Reports will be handled with confidentiality.

Maintainers may take action including:

- A warning
- Temporary suspension from community spaces
- Permanent removal from the project

## Scope

This Code of Conduct applies within all project spaces and in any public context where an individual is representing the XNET project.
